# Transformer

## 词嵌入和位置编码（token embedding / position embedding）

在 NLP 任务中，往往需要将自然语言的输入转化为机器可以处理的向量。

首先使用 tokenizer 分词器将语言序列处理为 token id 向量，再输入 embedding 层，所以 Embedding 内部其实是一个可训练的（Vocab_size，embedding_dim）的权重矩阵，词表里的每一个值，都对应一行维度为 embedding_dim 的向量。

例如：

```plain
"Hello world" (原始文本)
↓ [分词器处理]
[101, 7592, 2088, 102] (token IDs)
↓ [嵌入层处理]
[[0.1, 0.2, ..], [0.3, 0.4, ..], ...] (词向量)
```

由于 Transformer 独特的注意力机制，使得序列中相对位置丢失了（粗略理解一下，$QK^T$ 这样的矩阵乘法使得 token 的先后顺序无所谓了），这就需要位置编码来表示序列的相对位置。位置编码的方式很多，详情可以见下面这个链接：

[https://zhuanlan.zhihu.com/p/454482273](https://zhuanlan.zhihu.com/p/454482273)

Transformer 采用的是正余弦函数来进行位置编码（绝对位置编码 Sinusoidal），具体如下：

$$
\begin{cases}
PE(pos,2i)&=sin(pos/10000^{2i/d_{model}}) \\
PE(pos,2i+1)&=cos(pos/10000^{2i/d_{model}})
\end{cases}
$$

其中 pos 是 token 的绝对位置，i 为 1, 2, ..., d_model/2 - 1。

举例：

```plain
# 词向量示例 (4维)
I    = [0.3, 0.2, 0.5, 0.1]
love = [0.4, 0.3, 0.2, 0.6]
you  = [0.2, 0.4, 0.3, 0.5]

# pos=0（第一个词"I"的位置）
PE(0,0) = sin(0/10000^(0/4))  # 第0维
PE(0,1) = cos(0/10000^(0/4))  # 第1维
PE(0,2) = sin(0/10000^(2/4))  # 第2维
PE(0,3) = cos(0/10000^(2/4))  # 第3维

# pos=1（第二个词"love"的位置）
PE(1,0) = sin(1/10000^(0/4))  # 第0维
PE(1,1) = cos(1/10000^(0/4))  # 第1维
PE(1,2) = sin(1/10000^(2/4))  # 第2维
PE(1,3) = cos(1/10000^(2/4))  # 第3维

# pos=2（第三个词"you"的位置）
PE(2,0) = sin(2/10000^(0/4))  # 第0维
PE(2,1) = cos(2/10000^(0/4))  # 第1维
PE(2,2) = sin(2/10000^(2/4))  # 第2维
PE(2,3) = cos(2/10000^(2/4))  # 第3维

# 第一个词"I"的最终表示
I_final    = [0.3+0.0000, 0.2+1.0000, 0.5+0.0000, 0.1+1.0000]
# 第二个词"love"的最终表示
love_final = [0.4+0.8415, 0.3+0.5403, 0.2+0.0464, 0.6+0.9989]
# 第三个词"you"的最终表示
you_final  = [0.2+0.9093, 0.4-0.4161, 0.3+0.0927, 0.5+0.9957]
```

简单来说，PE(0,0) = sin(0/10000^(0/4)) 和 PE(0,1) = cos(0/10000^(0/4)) 这两个数，正好可以表示一个极坐标上的点，这样就让它和 PE(1,0) = sin(1/10000^(0/4))、PE(1,1) = cos(1/10000^(0/4)) 这两个可以表示同一个极坐标下的两个具有相对位置关系的不同点。

## 注意力机制

在《Attention is all you need》这篇文章中提出了著名的 **Transformer** 模型，Transformer 中抛弃了传统的 CNN 和 RNN，整个网络结构完全是由 Attention 机制组成。

$$Attention(Q,K,V) = Softmax(QK^T/\sqrt{d_k})V$$

矩阵 $Q, K, V$ 都源于输入特征本身，是根据输入特征分别乘上参数矩阵 $W_q, W_k, W_v$ 后的矩阵，但在工程上将其分化为：

- **$Q$**：要查询的信息（其实就是用这个矩阵去和其他输入特征的 $K$ 相乘算自己的注意力分数）
- **$K$**：被查询的向量（其实就是用这个矩阵被动地和其他人的 $Q$ 匹配算其他人的注意力分数）

这里的 $d_k$ 是特征向量维度，和 softmax 函数一样属于一种处理，核心还是 $Q$ 乘 $K$ 转置来计算 $Q$ 和 $K$ 的相似度，得到一个注意力分数，最后用注意力分数乘 $V$ 这个真值存放结果。

<img src="https://cdn.nlark.com/yuque/0/2025/png/57822622/1751454899287-22b9c98a-0f7f-4e01-bcde-856c6ceef60a.png" width="703" />

在经典的注意力机制中，$Q$ 往往来自于一个序列，$KV$ 来自于另一个序列，都通过参数矩阵计算得到，从而可以拟合这两个序列之间的关系。但在 Transformer 的 Encoder 结构中，使用的是注意力机制的变种 —— 自注意力（self-attention）机制。所谓自注意力，即是计算本身序列中每个元素对其他元素的注意力分布，即在计算过程中，$Q, K, V$ 都由同一个输入通过不同的参数矩阵计算得到，从而拟合输入语句中每一个 token 对其他所有 token 的关系。

同时，Transformer 还在 Decoder 使用了**掩码自注意力**（为了并行计算，生成一个上三角为掩码的矩阵）、**多头自注意力**（即同时对一个语料进行多次注意力计算，将最后的多次结果拼接起来作为最后的输出，即可更全面深入地拟合语言信息，本质在于有多组的参数矩阵 $W_q, W_k, W_v$）。

## Encoder：编码器

<img src="https://cdn.nlark.com/yuque/0/2025/png/57822622/1751966094802-60472e9a-6da8-4271-96d3-93a0b2336f86.png" width="752" />

一个 Encoder Layer 编码器由上面说的自注意力层和一个前馈神经网络组成，同时在自注意力层和前馈神经网络后面分别有一个**归一化层**，核心是为了让不同层输入的取值范围或者分布能够比较一致。由于深度神经网络中每一层的输入都是上一层的输出，因此多层传递下，对网络中较高的层，之前的所有神经层的参数变化会导致其输入的分布发生较大的改变。也就是说，随着神经网络参数的更新，各层的输出分布是不相同的，且差异会随着网络深度的增大而增大。但是，需要预测的条件分布始终是相同的，从而也就造成了预测的误差。

另外，由于 Transformer 模型结构较复杂、层数较深，为了避免模型退化，Transformer 采用了**残差连接**的思想来连接每一个子层。残差连接，即下一层的输入不仅是上一层的输出，还包括上一层的输入。残差连接允许最底层信息直接传到最高层，让高层专注于残差的学习。

## Decoder：解码器

和 Encoder 不同的是，Decoder 由两个注意力层和一个前馈神经网络组成。第一个注意力层是一个掩码自注意力层，即使用 Mask 的注意力计算，保证每一个 token 只能使用该 token 之前的注意力分数；第二个注意力层是一个多头注意力层，该层将使用第一个注意力层的输出作为 query，使用 Encoder 的输出作为 key 和 value，来计算注意力分数。

## 最后的线性层和 Softmax

线性层是一个简单的全连接神经网络，它将最终产生的矩阵投影到一个向量中，称为 logits 向量。假设我们的模型知道从训练数据集中学习到的 10,000 个独特的英语单词（我们模型的"输出词汇"）。这将使 logits 向量有 10,000 个单元格宽——每个单元格对应一个唯一单词的分数。然后，softmax 层将这些分数转化为概率（全部为正，全部加起来为 1）。选择概率最高的单元格，并生成与其关联的单词作为该时间步的输出。

# BERT —— Encoder only

BERT 的模型架构是取了 Transformer 的 Encoder 部分堆叠而成，是针对于 NLU（自然语言理解）任务打造的预训练模型，其输入一般是文本序列，而输出一般是 Label，例如情感分类的积极、消极 Label。但是，正如 Transformer 是一个 Seq2Seq 模型，使用 Encoder 堆叠而成的 BERT 本质上也是一个 Seq2Seq 模型，只是没有加入对特定任务的 Decoder，因此，为适配各种 NLU 任务，在模型的最顶层加入了一个分类头 prediction_heads，用于将多维度的隐藏状态通过线性层转换到分类维度（例如，如果一共有两个类别，prediction_heads 输出的就是两维向量）。

## 预训练

BERT 更大的创新点在于其提出的两个新的预训练任务上——MLM 和 NSP（Next Sentence Prediction，下一句预测）。预训练-微调范式的核心优势在于，通过将预训练和微调分离，完成一次预训练的模型可以仅通过微调应用在几乎所有下游任务上，只要微调的成本较低，即使预训练成本是之前的数倍甚至数十倍，模型仍然有更大的应用价值。

**MLM** 模拟的是"完形填空"。MLM 的思路也很简单，在一个文本序列中随机遮蔽部分 token，然后将所有未被遮蔽的 token 输入模型，要求模型根据输入预测被遮蔽的 token。

**NSP** 任务的核心思路是要求模型判断一个句对的两个句子是否是连续的上下文（在预训练任务上，有学者质疑 NSP 任务并不能提高模型性能，因为其太过简单，加入到预训练中并不能使下游任务微调时明显受益，甚至会带来负面效果）。

## 微调

作为 NLP 领域里程碑式的成果，BERT 的一个重大意义就是正式确立了预训练-微调的两阶段思想，即在海量无监督语料上进行预训练来获得通用的文本理解与生成能力，再在对应的下游任务上进行微调。

**所谓微调，其实和训练时更新模型参数的策略一致，只不过在特定的任务、更少的训练数据、更小的 batch_size 上进行训练，更新参数的幅度更小。**

## BERT 的问题

BERT 也存在一些问题，例如 MLM 任务和下游任务微调的不一致性，以及无法处理超过模型训练长度的输入等问题。

# T5 —— Encoder-Decoder

T5 模型的一个核心理念是"大一统思想"，即所有的 NLP 任务都可以统一为文本到文本的任务，这一思想在自然语言处理领域具有深远的影响。其设计理念是将所有不同类型的 NLP 任务（如文本分类、翻译、文本生成、问答等）转换为一个统一的格式：输入和输出都是纯文本。

例如：

- 对于文本分类任务，输入可以是 `classify: 这是一个很好的产品`，输出是 `正面`；
- 对于翻译任务，输入可以是 `translate English to French: How are you?`，输出是 `Comment ça va?`。

# LLM 的基础架构 —— Decoder only

现代大语言模型（如 GPT 系列）普遍采用 Decoder-only 架构，仅保留 Transformer 的 Decoder 部分，去掉 Encoder。这种架构的优势在于：

1. **自回归生成**：逐个 token 预测下一个词，天然适合文本生成任务
2. **上下文感知**：通过掩码自注意力机制，每个 token 只能看到之前的 token，保证了生成的因果性
3. **可扩展性强**：经过工程优化后可以训练千亿甚至万亿参数的模型

从 BERT 到 T5 再到 GPT，Transformer 架构的演变推动了整个 NLP 领域的范式转移——从特定任务微调走向通用大模型。
